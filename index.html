Sistema de Reserva del Hotel Veuxor

Bienvenidos al sistema de reservas del Hotel Veuxor, una herramienta integral diseñada para optimizar la gestión de reservas y servicios en nuestro prestigioso establecimiento. El Hotel Veuxor se enorgullece de ofrecer a sus huéspedes una experiencia de estancia inigualable, y nuestro sistema de reservas es un reflejo de nuestro compromiso con la excelencia y la innovación.

Sobre nuestro proyecto

En este proyecto se uso la técnica de Programación Orienta en Objetos (POO), la cual permite tener una estructura clara y mantenible del código del sistema de hotel, mientras que MySQL proporciona un medio eficiente para gestionar y consultar los datos necesarios para las operaciones del hotel Veuxor.

Informacion

Función del sistema

Este sistema está orietado en la reserva de una habitación, donde se puede navegar por las diversas habitaciones que estén disponibles, donde al final se hará una facturación de la habitación que ha solicitado un cliente para su estadía.

Habitaciones

En el sistema de gestión de hotel, la navegación para seleccionar y reservar una habitación está diseñada para ser intuitiva y eficiente, facilitando a los clientes encontrar la opción de alojamiento ideal según sus preferencias y necesidades. En este sistema se tiene la reserva de habitaciones ya se de tipo Personal, Doble, Matrimonial o Triple.

Facturación

En el sistema de gestión de hotel, la facturación del pedido de una habitación es un proceso crucial que asegura la correcta contabilización y cobro de los servicios prestados a los huéspedes. Cuando un cliente realiza una reserva de habitación, el sistema genera automáticamente una factura que detalla todos los cargos asociados durante su estancia.

Informacion del sistema

La creación de este sistema trae consigo claras ventajas y desventajas al implementar la gestión de un hotel, es este caso el hotel Veuxor, utilizando POO y MySQL, lo que es crucial para tomar decesiones informadas sobre el desarrollo y uso.

Tabla Comparativa

Uso de POO

Pros Contras
Reutilización Facilita la creación de módulos reutilizables y mantenibles. Puede requerir más tiempo y esfuerzo inicial para diseñar la estructura.
Encapsulamiento Protege los datos y garantiza la integridad mediante métodos de acceso. Puede aumentar la complejidad del código debido a la abstracción.
Herencia y Polimorfismo Permite la reutilización de código y la creación de jerarquías lógicas. Puede llevar a diseños complicados y difíciles de seguir.
Mantenimiento Facilita la localización y corrección de errores. El mantenimiento puede ser complicado si el sistema se vuelve muy grande.
Tabla Comparativa

Uso de MySQL

Pros Contras
Rendimiento y Escalabilidad MySQL es eficiente para manejar grandes volúmenes de datos y múltiples usuarios. El rendimiento puede disminuir con consultas complejas en bases de datos muy grandes.
Integridad de Datos Asegura la integridad mediante claves primarias, foráneas y restricciones. La integridad puede verse comprometida si no se diseñan correctamente las relaciones y restricciones.
Facilidad de Uso y Acceso Amplia documentación y herramientas de gestión disponibles. Puede ser menos intuitivo para usuarios sin experiencia en bases de datos relacionales.
Costo MySQL es de código abierto y gratuito. Puede haber costos asociados con soporte y servicios adicionales.
